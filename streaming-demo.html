PROCASSEF Streaming Updates Demo

This page demonstrates how to implement real-time streaming updates for the PROCASSEF dashboard.

How It Works

The streaming update system allows you to smoothly update charts and tubes in real-time without requiring a full page refresh or complete chart recreation.

Two primary update methods are supported:

  • WebSocket Updates: For true real-time data with minimal latency
  • Polling Updates: For regular updates at specified intervals

The updates are applied in a way that minimizes visual disruption while ensuring data accuracy.

Implementation Examples

WebSocket Example

// Connect to a WebSocket stream streamingUpdateService.connectToStream('wss://your-api.example/dashboard-stream', { useWebSockets: true, autoReconnect: true }); // Listen for events streamingUpdateService.on('streamConnected', () => { console.log('WebSocket connected!'); }); streamingUpdateService.on('dataReceived', (data) => { console.log('New data received:', data); });

Polling Example

// Connect to a polling endpoint streamingUpdateService.connectToStream('https://your-api.example/dashboard-data', { useWebSockets: false, pollingInterval: 30000, // 30 seconds autoReconnect: true });

Manual Update Example

// Update with custom data const newData = { kpis: { completionRate: 76, qualityScore: 82, efficiencyScore: 78, ctasfRate: 65, dailyCompletion: 85, dailyTarget: 100, dailyCompletionRate: 85, weeklyCompletion: 450, weeklyTarget: 500, weeklyCompletionRate: 90, monthlyCompletion: 1800, monthlyTarget: 2000, monthlyCompletionRate: 90 }, // Add any other data needed for chart updates streamingUpdate: true }; streamingUpdateService.updateDashboard(newData);
